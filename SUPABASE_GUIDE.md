# دليل تعديل نظام إخلاء الطرف لـ Supabase

## تمهيد
هذا الدليل يشرح كيفية نقل النظام من `localStorage` إلى قاعدة بيانات Supabase السحابية، وكيفية التعديل على النظام لاحقاً.

---

## أولاً: إعداد Supabase (مرة واحدة)

### 1. إنشاء حساب ومشروع
1. ادخل إلى [supabase.com](https://supabase.com)
2. اضغط "Start your project"
3. سجل دخول بـ GitHub أو Google
4. اضغط "New Project"
5. اختر اسماً للمشروع (مثلاً: diba-clearance)
6. أدخل كلمة مرور قوية لقاعدة البيانات
7. اختر المنطقة الأقرب (Singapore أو Tokyo)
8. اضغط "Create new project" وانتظر 2 دقيقة

### 2. إنشاء جداول قاعدة البيانات
بعد إنشاء المشروع، اضغط على "SQL Editor" من القائمة اليسرى، ثم "New Query" والصق الكود التالي:

```sql
-- جدول المستخدمين
CREATE TABLE users (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  username TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  full_name TEXT NOT NULL,
  email TEXT,
  department TEXT,
  role TEXT DEFAULT 'employee',
  is_active BOOLEAN DEFAULT true,
  last_login TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- جدول الأقسام
CREATE TABLE departments (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name TEXT NOT NULL,
  description TEXT,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW()
);

-- جدول طلبات إخلاء الطرف
CREATE TABLE clearance_requests (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  request_number TEXT UNIQUE NOT NULL,
  employee_name TEXT NOT NULL,
  employee_id TEXT NOT NULL,
  created_by TEXT NOT NULL,
  department TEXT NOT NULL,
  position TEXT,
  phone TEXT,
  email TEXT,
  reason TEXT,
  status TEXT DEFAULT 'pending',
  current_step INTEGER DEFAULT 1,
  total_steps INTEGER DEFAULT 5,
  created_at TIMESTAMP DEFAULT NOW(),
  steps JSONB DEFAULT '[]',
  replies JSONB DEFAULT '[]'
);

-- إدراج بيانات تجريبية (اختياري)
INSERT INTO departments (name, description) VALUES
  ('الموارد البشرية', 'إدارة شؤون الموظفين'),
  ('الشؤون المالية', 'الإدارة المالية'),
  ('تقنية المعلومات', 'قسم IT'),
  ('المكتبة الطبية', 'إدارة المراجع والكتب الطبية'),
  ('إدارة المرافق', 'صيانة وإدارة المرافق');

-- مستخدم مدير افتراضي
INSERT INTO users (username, password, full_name, role, department, is_active)
VALUES ('admin', 'admin123', 'مدير النظام', 'admin', 'الإدارة', true);
```

اضغط "Run" لتنفيذ الكود.

### 3. الحصول على بيانات الاتصال
1. اضغط على "Project Settings" (أيقونة الترس) من القائمة اليسرى
2. اضغط "API" من القائمة الفرعية
3. انسخ "Project URL" (يبدأ بـ https://...)
4. انسخ "anon/public" key (مفتاح طويل جداً)

## ثانياً: تعديل الملف `standalone.html`

### 1. تعديل إعدادات Supabase في الملف
افتح الملف `standalone.html` وابحث عن هذه الأسطر في أعلى الملف:

```javascript
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```

استبدلها ببياناتك:
```javascript
const SUPABASE_URL = 'https://xxxx.supabase.co'; // ضع رابط مشروعك
const SUPABASE_ANON_KEY = 'eyJhbGc...'; // ضع المفتاح الخاص بك
```

### 2. رفع الملفات (التخزين)
إذا أردت رفع ملفات حقيقية (وليس كـ Base64)، فعل Supabase Storage:
1. اضغط "Storage" من القائمة اليسرى
2. اضغط "Create bucket"
3. سمّه `files` واجعله public
4. عدل دالة `submitReply` في الملف لترفع للـ Storage بدلاً من حفظ Base64

## ثالثاً: كيفية التعديل على النظام لاحقاً

### سيناريو: تعديل بسيط (تغيير ألوان، نصوص، إضافة حقول)

1. **عدل الملف محلياً** (`C:\Users\HP\Desktop\Pro1\standalone.html`)
2. **اختبر التعديل** بفتح الملف في المتصفح
3. **ارفع النسخة المحدثة** للاستضافة

### سيناريو: تعديل في قاعدة البيانات (إضافة جدول، حقل جديد)

1. ادخل إلى Supabase > SQL Editor
2. نفذ أمر SQL لإضافة الحقل، مثلاً:
   ```sql
   ALTER TABLE clearance_requests ADD COLUMN notes TEXT;
   ```
3. عدل الملف `standalone.html` ليتعامل مع الحقل الجديد

### سيناريو: رفع نسخة محدثة للاستضافة

#### إذا استخدمت Vercel:
```bash
# في مجلد المشروع
git add standalone.html
git commit -m "تحديث النظام"
git push
# Vercel ستنشر التحديث تلقائياً
```

#### إذا استخدمت Render:
- ارفع الملف المحدث عبر واجهة الموقع، أو
- اربط GitHub وارفع التعديلات عبر Git

#### إذا استخدمت استضافة عادية (FTP):
- ارفع `standalone.html` عبر برنامج FTP (مثل FileZilla)

## رابعاً: نصائح هامة

### ✅ افعل هذه:
- احتفظ دائماً بنسخة محلية من الملف قبل التعديل
- جرب التعديل محلياً أولاً قبل الرفع
- أنشئ "Backup" لقاعدة البيانات دورياً من Supabase > Database > Backups

### ❌ لا تفعل هذه:
- لا تشارك مفتاح `service_role` مع أي شخص
- لا تحذف الجداول من Supabase إلا إذا كنت متأكداً
- لا ترفع ملفات ضخمة كـ Base64 (استخدم Storage بدلاً منها)

## خامساً: استكشاف الأخطاء

### المشكلة: "فشل الاتصال بـ Supabase"
**الحل:** تأكد أن الرابط والمفتاح صحيحان، وتأكد أن المشروع "Active" في Supabase

### المشكلة: "البيانات لا تظهر"
**الحل:** افتح Developer Tools (F12) > Console وابحث عن أخطاء

### المشكلة: "خطأ في رفع الملف"
**الحل:** تأكد من حجم الملف (الحد الأقصى 5MB في النسخة الحالية)

3. اختر "Static Site"
4. ارفع `standalone.html`
5. ستحصل على رابط مثل: `https://diba-clearance.vercel.app`

## سابعاً: التراجع عن التعديلات (Rollback)

إذا أردت العودة للنسخة القديمة (localStorage فقط):
1. افتح `standalone.html`
2. غيّر القيم في الأعلى إلى:
```javascript
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```
3. النظام سيعود تلقائياً لاستخدام `localStorage`
